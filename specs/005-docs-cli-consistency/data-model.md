# Data Model: Docs-CLI Consistency

**Feature**: 005-docs-cli-consistency | **Date**: 2026-03-07

## Overview

This feature's data model is not about new database entities — it's about documenting the **authoritative schemas** that all documentation must reference correctly. These schemas already exist in Go code. This document serves as the single reference for what the docs must match.

## Entity 1: CLI Command

**Source of truth**: `cmd/apx/commands/root.go` and individual command files

A command is registered via `cobra.Command` with these documentation-relevant properties:

| Field | Type | Description | Example |
|-------|------|-------------|---------|
| `Use` | string | Command name + arg pattern | `"lint [path]"` |
| `Short` | string | One-line description | `"Lint API schemas"` |
| `Long` | string | Extended description | Multi-line help text |
| `Args` | validator func | Positional arg constraints | `cobra.MaximumNArgs(1)` |
| `Flags` | FlagSet | Named flags | `--format`, `--against` |
| `RequiredFlags` | []string | Flags marked required | `["against"]` |

## Entity 2: Config File (`apx.yaml`)

**Source of truth**: `internal/config/config.go` → `Config` struct

### Schema

```yaml
# apx.yaml — v1 schema
version: 1                     # int, required, must be non-zero
org: "myorg"                   # string, required, must be non-empty
repo: "apis"                   # string, required, must be non-empty
module_roots:                  # []string, optional
  - "proto"
  - "openapi"
language_targets:              # map[string]LanguageTarget, optional
  go:
    output: "internal/gen/go"
    options:
      paths: "source_relative"
policy:                        # Policy, optional
  breaking:
    against: "main"
publishing:                    # PublishingConfig, optional
  canonical_repo: "github.com/myorg/apis"
  subtree_prefix: "proto"
tools:                         # ToolsConfig, optional
  buf:
    version: "1.47.2"
execution:                     # ExecutionConfig, optional
  container:
    image: "bufbuild/buf"
    tag: "1.47.2"
```

### Validation Rules

1. `version` must be non-zero integer
2. `org` must be non-empty string
3. `repo` must be non-empty string

### Known Bug (to fix)

`apx init app` generates:
```yaml
kind: proto           # NOT in Config struct
module: payments.v1   # NOT in Config struct
org: testorg
version: v1           # WRONG TYPE — string, not int
                      # MISSING: repo (required)
```

**Fix**: Change `generateApxYaml` to produce Config-compatible YAML with `version: 1`, `org`, `repo`, and valid fields.

## Entity 3: Lock File (`apx-lock.yaml`)

**Source of truth**: `internal/config/dependencies.go` → `LockFile` struct

### Schema

```yaml
# apx-lock.yaml — v1 schema
version: 1                              # int
toolchains:                             # map[string]ToolchainLock
  buf:
    version: "1.47.2"
    checksum: "sha256:abc123..."
dependencies:                           # map[string]DependencyLock
  "proto/payments/ledger/v1":           # key = module path
    repo: "github.com/myorg/apis"      # string
    ref: "v1.2.3"                       # string (git ref)
    modules:                            # []string
      - "proto/payments/ledger/v1"
```

### Notes for docs

- Dependencies is a **map** keyed by module path, not a flat list
- Each `DependencyLock` has `Repo`, `Ref`, `Modules` fields (not `version`, `source`, etc.)
- Lock file is managed by `DependencyManager.Add/Remove/List` — not manually edited

## Entity 6: Exit Codes

| Code | Meaning | Produced By |
|------|---------|-------------|
| 0 | Success | All commands on success |
| 1 | General error | Cobra default, command failures |
| 6 | Schema validation failure | `lint`, `breaking`, `policy check` |

## Relationships

```
apx.yaml ─── loaded by ──→ Config struct ─── used by ──→ All commands
                                                           │
apx-lock.yaml ─ loaded by → LockFile struct ─ used by ──→ add, sync, unlink, gen
                                                           │
catalog.yaml ── loaded by → Catalog struct ── used by ──→ search, catalog build
                                                           │
CLI flags ───── parsed by → Cobra ──────────── override → Config fields
```
