# Contract: Doc-Parity Test Suite

**Feature**: 005-docs-cli-consistency | **Date**: 2026-03-07

## Purpose

Define what the doc-parity test suite must verify to ensure documentation stays in sync with the CLI binary. This contract specifies the test categories, their inputs, and expected behaviors.

## Contract 1: Command Existence Tests

**Location**: `cmd/apx/commands/doc_parity_test.go`

### Invariant

Every command name mentioned in any `docs/*.md` file MUST be resolvable via `root.Find([]string{...})`.

### Test Pattern

```go
func TestDocParity_AllCommandsExist(t *testing.T) {
    root := NewRootCmd("test")
    // For each command documented in CLI reference:
    commands := [][]string{
        {"init"}, {"init", "canonical"}, {"init", "app"},
        {"lint"}, {"breaking"},
        {"semver"}, {"semver", "suggest"},
        {"gen"}, {"policy"}, {"policy", "check"},
        {"catalog"}, {"catalog", "build"},
        {"publish"}, {"search"}, {"add"},
        {"sync"}, {"unlink"},
        {"config"}, {"config", "init"}, {"config", "validate"},
        {"fetch"}, {"completion"},
    }
    for _, args := range commands {
        cmd, _, err := root.Find(args)
        // ASSERT: err == nil && cmd.Use != "apx"
    }
}
```

### Coverage

- All 14 registered commands + 4 subcommands + `completion`
- Both parent commands (semver, policy, catalog, config) and their children

## Contract 2: Flag Existence Tests

**Location**: `cmd/apx/commands/doc_parity_test.go`

### Invariant

Every flag documented for a command MUST exist in that command's `FlagSet`.

### Test Pattern

```go
type flagSpec struct {
    command []string
    flags   []string
}

var documentedFlags = []flagSpec{
    {[]string{"breaking"}, []string{"against", "format"}},
    {[]string{"semver", "suggest"}, []string{"against"}},
    {[]string{"gen"}, []string{"out", "clean", "manifest"}},
    {[]string{"publish"}, []string{"module-path", "canonical-repo", "version", "dry-run", "create-pr"}},
    {[]string{"search"}, []string{"format", "catalog"}},
    {[]string{"sync"}, []string{"clean", "dry-run"}},
    {[]string{"fetch"}, []string{"config", "output", "verify"}},
    {[]string{"lint"}, []string{"format"}},
    {[]string{"init", "canonical"}, []string{"org", "repo", "skip-git", "non-interactive"}},
    {[]string{"init", "app"}, []string{"org", "non-interactive"}},
}
```

### Coverage

- Every flag listed in data-model.md per-command table
- Both local and inherited (persistent) flags

## Contract 3: Required Flag Tests

**Location**: `cmd/apx/commands/doc_parity_test.go`

### Invariant

If a flag is marked required in code (via `MarkFlagRequired`), documentation MUST show it as required and include it in all examples that use the command.

### Test Pattern

```go
// When --against is omitted from "breaking", execution must fail
cmd.SetArgs([]string{"breaking", "."})
err := cmd.Execute()
// ASSERT: err != nil (required flag "against" not set)
```

### Coverage

- `breaking --against` (required)
- `semver suggest --against` (required)

## Contract 4: Argument Count Tests

**Location**: `cmd/apx/commands/doc_parity_test.go`

### Invariant

Documentation examples MUST NOT pass more arguments than the command's `Args` validator allows.

## Contract 5: Config Roundtrip Tests

**Location**: `cmd/apx/commands/doc_parity_test.go`

### Invariant

Config files generated by `apx init` MUST pass `apx config validate`.

### Test Pattern

```go
// 1. Run apx init canonical → generates apx.yaml
// 2. Load the generated apx.yaml with config.Load()
// 3. ASSERT: no error
// Repeat for apx init app
```

### Coverage

- `init canonical` → apx.yaml roundtrip
- `init app` → apx.yaml roundtrip
- `apx.example.yaml` → validate roundtrip

## Contract 6: Golden Help Tests

**Location**: `testdata/golden/help/` + testscript or Go test

### Invariant

The `--help` output of the root command matches the golden file. This prevents silent command additions/removals from going undocumented.

### Current State

- `testdata/golden/help/main.golden` exists and is tested
- Individual command golden files do NOT exist yet (not required for this feature, but recommended as future work)

## Contract 7: "Planned" Marker Consistency

**Location**: Manual review + Sphinx build

### Invariant

Commands marked as "Planned" in docs MUST NOT exist in the binary. Conversely, commands that exist in the binary MUST NOT be marked as "Planned".

### Verification

Run a Sphinx build (`make -C docs html`) and confirm zero broken-link warnings. Separately, a test can grep doc files for "Planned" admonitions and cross-reference against the registered command list.

## Contract 8: Documentation Build

**Location**: CI / local Sphinx build

### Invariant

`sphinx-build -W -b html docs docs/_build` (warnings-as-errors) MUST exit 0.

### Coverage

- All toctree references resolve
- No broken cross-references
- All admonition directives are well-formed

## Test Execution

All doc-parity tests run as part of `go test ./cmd/apx/commands/...` (no special tags needed). They do NOT require E2E_ENABLED since they test the in-process cobra command tree and don't invoke external binaries.

The Sphinx build check (Contract 8) runs as part of the docs CI job, not the Go test suite.
