feat(008): implement external API registration as first-class feature

Complete implementation of feature 008-external-api-registration with full
end-to-end functionality and comprehensive testing.

## What This Implements

Enables platform teams to register third-party APIs alongside first-party APIs
with full provenance tracking, dependency integration, versioning, and lifecycle
management.

## New Components

### Core Types
- ExternalRegistration struct with CRUD operations, validation, and persistence
- ExternalAPIs list in Config struct + DependencyLock provenance fields
- Module provenance fields (Origin, ManagedRepo, UpstreamRepo, UpstreamPath, ImportMode)

### CLI Commands
- `apx external register` - register external APIs with upstream tracking
- `apx external list` - list registered APIs with optional origin filtering
- `apx external transition` - transition between external and forked classification

### Enhanced Commands  
- `apx search --origin` - filter APIs by origin (first-party, external, forked)
- `apx show` - displays Provenance section for external APIs
- `apx inspect identity` - shows Origin/Import/Managed/Upstream lines
- `apx add` - records provenance in lock file for external API dependencies
- `apx catalog generate` - merges external APIs into catalog with conflict detection

## Implementation Details

### Five User Stories (All Complete)
- US1: Register external APIs with managed/upstream coordinates
- US2: Discover and inspect external APIs with full provenance
- US3: Add external APIs as dependencies with lock file provenance
- US4: Transition between external and forked classifications
- US5: Version and lifecycle management identical to first-party APIs

### Type System
- 12-field ExternalRegistration with sensible defaults (preserve, external)
- Conflict detection: API ID and path conflicts with first-party modules
- Version tracking: stable vs prerelease separation via semver
- Lifecycle states: experimental, beta, stable, deprecated, sunset

### Validation
- API ID format enforcement (format/domain/name/line)
- Repository URL validation
- Managed path validation (no leading/trailing slashes, no traversal)
- Import mode and origin enum validation

### Testing
- 25/25 testscripts PASS (all 5 new + all 20 pre-existing)
- 8/8 internal package test suites PASS
- Unit tests: ExternalRegistration CRUD, MergeExternalAPIs, origin filtering
- Integration tests: register, search, show, inspect, dep add, version, transition
